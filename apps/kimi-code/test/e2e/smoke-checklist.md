# Kimi Code TUI 冒烟测试清单（agent-core-v2 回归版）

## 用途

TUI 接入 `agent-core-v2` 后的全功能回归冒烟清单。覆盖：启动与 UI、对话流、全部内置工具、三种权限模式 + plan 模式、全部斜杠命令、编辑器交互、resume / replay / 会话管理、goal / swarm / 后台任务、设置与配置、非交互模式（`kimi -p`）与 CLI 子命令、终端兼容性。

## 如何使用

1. 按「第 0 章」准备环境，全程在隔离配置目录 + 测试工作区里执行，不要污染日常配置。
2. 逐条执行用例，通过即在 `- [ ]` 打勾；失败则按附录 B 的模板记录（用例编号 + 现象 + 复现步骤 + 截图/日志）。
3. 标 ★ 的用例是 P0 核心路径（约 15 分钟可跑完），时间紧张时至少跑完这些；完整回归约需 60–90 分钟。
4. 每条用例都验证两个维度：**预期 UI**（渲染对不对）和 **预期行为**（Agent 做得对不对），两者都通过才算通过。

## 0. 测试前准备

### 0-1 构建
- [ ] **操作**：在仓库根目录执行 `pnpm install && pnpm build`（或用 `pnpm dev:cli` 走开发态）。
- **预期**：构建无错误；`node apps/kimi-code/dist/main.mjs --version` 输出与 `apps/kimi-code/package.json` 中版本一致。

### 0-2 隔离配置目录
- [ ] **操作**：
  ```bash
  export KIMI_CODE_HOME=/tmp/kimi-smoke/home
  rm -rf /tmp/kimi-smoke && mkdir -p "$KIMI_CODE_HOME"
  ```
  之后所有 `kimi` 命令都在这个环境下执行。
- **预期**：冒烟产生的会话、配置、登录态全部落在 `/tmp/kimi-smoke/home`，不影响日常 `~` 下的配置。

### 0-3 测试工作区（fixture）
- [ ] **操作**：
  ```bash
  export SMOKE_DIR=/tmp/kimi-smoke/ws
  mkdir -p "$SMOKE_DIR/src" "$SMOKE_DIR/assets" && cd "$SMOKE_DIR"
  git init -q
  echo "# Smoke Fixture" > README.md
  printf 'hello\n' > src/hello.txt
  printf 'export const a = 1;\n// TODO: refine this\n' > src/a.ts
  printf 'line one\nline two\nline three\n' > src/multi.txt
  # 手工放入任意一张 png 图片到 assets/pic.png
  ```
- **预期**：工作区是一个 git 仓库，含 markdown、txt、ts 文件和一张图片。

### 0-4 凭证
- [ ] **操作**：首次启动后按提示完成登录（OAuth 设备码或 API key），或用 `/login`。
- **预期**：能正常发出第一轮对话；后续用例默认已登录。
- **注意**：`/logout` 用例（9-12）安排在最后执行，避免中途清掉凭证。

### 约定
- 下文「发送 X」均指在 TUI 输入框输入 X 并回车。
- 审批弹窗出现时，默认选择「批准/允许」类选项，除非用例另有说明。
- 观察状态栏时关注三要素：当前模型、权限模式、上下文用量。

## 2. 基础对话流（Agent 核心行为）

### 2-1 单轮问答 ★
- [ ] **操作**：发送「只回复一个词：hi」。
- **预期 UI**：输入框清空并显示加载动画；回复逐字流式输出，Markdown 正常渲染；turn 结束后回到可输入状态，状态栏上下文用量上升。
- **预期行为**：恰好一个 turn，无多余工具调用，回复内容为单个词。

### 2-2 多轮上下文
- [ ] **操作**：先发「记住数字 42，只回复『好』」，再发「我刚才让你记住的数字是多少？」。
- **预期行为**：第二轮正确回答 42，证明上下文在多轮间保留。

### 2-3 思考过程展示
- [ ] **操作**：`/effort`（别名 `/thinking`）切换到高档位，发送「一步一步推理：17 × 23 等于多少？」。
- **预期 UI**：如模型支持，回复中出现 thinking/推理块，且与正文有明确视觉区分；状态栏 effort 标识同步更新。

### 2-4 中断生成 ★
- [ ] **操作**：发送「写一篇 500 字短文」，生成过程中按 Ctrl-C（或 Esc）。
- **预期 UI**：流式输出立即停止，界面回到可输入状态，已输出内容保留在 transcript 中。
- **预期行为**：可以立刻发下一条消息继续对话，无卡死、无残留加载动画。

### 2-5 消息排队
- [ ] **操作**：发送「慢慢数从 1 到 20」后，趁生成中再输入「停一下，改数到 5」并回车；生成中在空输入框按 ↑。
- **预期 UI**：第二条消息进入队列面板（queue pane）显示，不插队打断当前 turn；按 ↑ 后队尾消息被召回输入框可重新编辑。
- **预期行为**：当前 turn 结束后，队列中的消息自动依次发送执行。

### 2-6 Steer（生成中插话）
- [ ] **操作**：发送「详细讲讲操作系统」生成中按 Ctrl-S，输入「用一句话总结即可」并确认。
- **预期 UI**：steer 输入框正常弹出与提交。
- **预期行为**：当前回复按插话内容调整方向（明显收短）。

### 2-7 错误处理
- [ ] **操作**：`/model` 切到一个不可用/未配置的模型发一条消息；随后切回可用模型再发一条。
- **预期 UI**：失败 turn 显示清晰的错误信息（非堆栈糊屏、非静默卡死）。
- **预期行为**：切回可用模型后对话正常恢复，会话不损坏。

### 2-8 空输入保护
- [ ] **操作**：直接回车；输入纯空格回车。
- **预期行为**：均不发起 turn，无空消息进入 transcript。

### 3-11 后台任务（TaskList / TaskOutput / TaskStop）
- [ ] **操作**：发送「开一个后台任务，慢慢把 README.md 逐字翻译成英文，期间我们聊别的」；随后随便发一条无关消息；中途执行 `/tasks` 打开任务浏览器，进入查看输出；再发「停掉那个后台翻译任务」。
- **预期 UI**：后台任务在界面有状态提示（运行中/完成/已停止）；`/tasks` 浏览器列出任务、可选中查看实时输出（task-output-viewer）；停止后状态更新。
- **预期行为**：后台任务与前台对话并行不阻塞；停止后不再产出输出。

### 3-12 Skill
- [ ] **操作**：确认环境中至少有一个可用 skill（内置或 `~/.kimi-code/skills` 下安装），发送「用 <某个可用 skill 名> 技能完成一件小事，并告诉我你调用了什么」。
- **预期 UI**：出现 Skill 工具调用块，展示技能名称与参数。
- **预期行为**：技能内容被加载并据此回答，不胡编；环境无任何 skill 时改用「列出你可用的技能」验证降级提示。

### 3-13 AskUser（Agent 反问）
- [ ] **操作**：发送「在动手之前，先问我一个带 2~3 个选项的问题，确认你希望我用哪种格式给你输出」。
- **预期 UI**：弹出 question-dialog，展示问题与编号选项，支持方向键/数字键选择，也支持自定义输入；选择后对话框关闭。
- **预期行为**：Agent 拿到选择结果后继续，且后续输出符合所选格式。

### 3-14 图片读取（ReadMediaFile）
- [ ] **前置**：需使用支持视觉的模型——ReadMediaFile 按模型媒体能力动态注册，无视觉能力的模型下工具不注册（属预期门控）。
- **操作**：发送「看一下 assets/pic.png，描述画面内容」。
- **预期 UI**：媒体工具调用块正常展示（图片不糊屏）。
- **预期行为**：描述与图片实际内容大体一致；若当前模型无视觉能力，验证 Agent 明确告知工具不可用而非编造描述。

### 3-15 MCP（可选）
- [ ] **前置**：配置过一个 MCP server 时执行；未配置则验证 `/mcp` 显示空状态后跳过。

- **预期 UI**：MCP 工具与普通工具渲染一致，能区分来源。
- **预期行为**：调用成功并返回真实数据。

## 4. 权限模式与审批

### 4-1 manual 模式审批流 ★
- [ ] **操作**：确认处于 manual（状态栏/`/status` 可见）。发送「运行 echo hello 命令」触发 Bash 审批；弹窗出现时先选「拒绝」；再发一次同样指令，这次选「批准」。
- **预期 UI**：审批面板展示工具名、目标命令/路径与编号选项（Approve once / Approve for this session / Reject / Reject with feedback）；支持数字键与方向键选择、Esc 拒绝。
- **预期行为**：拒绝后命令未执行，Agent 报告被拒绝并停下或征求下一步；批准后命令真实执行。
- **备注（0.23.3 实测）**：manual 下工作区内的 Write/Edit **不弹审批**直接执行，仅 Bash 类命令被拦截——若与 "Approve every action yourself" 的设计预期不符，按回归问题跟踪。

### 4-2 始终允许类选项
- [ ] **操作**：审批弹窗中如有「始终允许/不再询问」类选项，选中后再让 Agent 做一次同类写操作。
- **预期行为**：同类操作后续不再弹审批，直接执行；状态/规则在会话内生效。

### 4-3 yolo 模式 ★
- [ ] **操作**：执行 `/yolo`（别名 `/yes`）切到 yolo。发送「创建 tmp-yolo.md 写一行 ok」；再发送「删除 src/hello.txt」。
- **预期 UI**：状态栏权限模式同步变为 yolo。
- **预期行为**：普通写操作免审批直接执行；删除文件这类风险操作仍弹审批询问。

### 4-4 auto 模式
- [ ] **操作**：执行 `/auto` 切到 auto。发送「删除 src/hello.txt 并创建 gone.md 写一行 ok」。
- **预期 UI**：状态栏同步变为 auto。
- **预期行为**：包括删除在内的所有操作全程无审批自动执行；文件变更真实发生。

### 4-5 /permission 选择器
- [ ] **操作**：执行 `/permission`，在 manual / yolo / auto 间各切一次。
- **预期 UI**：选择器展示三档及说明；切换后状态栏立即同步。
- **预期行为**：切换即时生效（下一次工具调用按新模式执行）。

### 4-6 /yolo 与 /auto 互斥切换
- [ ] **操作**：yolo 状态下执行 `/auto`，再执行 `/yolo`。
- **预期行为**：两档互斥，状态栏始终只显示一种模式，无叠加错乱。

### 4-7 plan 模式 ★
- [ ] **操作**：按 Shift-Tab（或 `/plan`）进入 plan 模式；发送「在 src 下新增一个 add.ts 导出加法函数，并说明怎么做」；计划出现后批准执行；执行完成后再按 Shift-Tab（或 `/plan clear`）退出。
- **预期 UI**：状态栏显示 plan 标识；Agent 进行只读探索后通过 ExitPlanMode 弹出计划审批面板，展示完整计划；批准后进入执行。
- **预期行为**：探索阶段无任何写操作（不写文件、不跑修改类命令）；批准后才开始真实写入 `src/add.ts`；`/plan clear` 后 plan 状态彻底清除。

### 4-8 模式跨会话行为
- [ ] **操作**：在 yolo 下 `/new` 开新会话；再退出 `kimi` 重新进入。
- **预期行为**：记录新会话/重启后的权限模式（保持或回落默认），行为前后一致、可解释，不出现状态栏与实际审批行为不一致的情况。

## 7. 会话管理：resume / replay / fork / undo / 导出
